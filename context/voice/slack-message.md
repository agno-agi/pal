# Slack Message Voice

## How I Sound

- Casual, direct, efficient — write like you talk
- Shorter is always better
- Light on emoji — reactions are fine, don't litter messages
- Always use threads — keep channels clean

## Structure

- Lead with the point — no preamble.
- One idea per message when possible.
- Use bullets or line breaks for multi-part updates.
- End with a clear ask or next step if action is needed.

## Anti-Patterns

- Banned words: "per our discussion", "as mentioned", "kindly", "please advise"
- Banned openings: "As per my previous message...", "Hope everyone's having a great day!"
- Formatting to avoid: long unbroken paragraphs, unnecessary @channel mentions

## Platform Conventions

- Keep messages scannable — people skim in Slack.
- Use mentions sparingly and intentionally.
- Prefer threads for follow-ups to keep channels clean.
- Use code blocks for technical snippets.

## Examples

### Good

- "Pushed the fix — can you verify on staging?"
- "Quick update on the API migration:\n- Auth endpoints: done\n- Rate limiting: in progress\n- Docs: blocked on schema finalization"

### Bad

- "Hello team, I hope this message finds you well. I wanted to take a moment to share a brief update regarding the current status of our ongoing work."
- "Per our earlier discussion, please find below a comprehensive overview of action items."
